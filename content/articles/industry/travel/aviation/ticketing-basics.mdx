## Введение

Выписка авиабилетов — фундаментальный навык для каждого специалиста туристической отрасли. Понимание структуры бронирования, процессов выписки, переоформления и возврата билетов напрямую влияет на качество обслуживания клиентов и финансовые результаты компании.

Для сотрудников Otrar Travel владение этими знаниями особенно важно: ошибки в оформлении билетов приводят к финансовым потерям, недовольству клиентов и репутационным рискам. Данный материал закладывает основы профессиональной работы с авиабилетами.

## Ключевые понятия

### PNR structure

PNR (Passenger Name Record) — запись данных о пассажире в системе бронирования. Это уникальный буквенно-цифровой код, идентифицирующий бронирование.

**Обязательные элементы PNR:**

1. **Name element (имя пассажира)**
   - Формат: ФАМИЛИЯ/ИМЯ MR или MS
   - Должно точно соответствовать документу
   - Пример: `IVANOV/ALEXEY MR`

2. **Itinerary (маршрут)**
   - Сегменты перелёта с датами и рейсами
   - Статус сегмента (HK — подтверждён, HL — лист ожидания)
   - Пример: `1. KC 871 Y 15DEC ALAFRA HK1 0820 1125`

3. **Contact element (контактная информация)**
   - Телефон агентства и пассажира
   - Email для уведомлений
   - Пример: `CTCM +7 776 532 99 77`

4. **Ticketing element (срок выписки)**
   - Дедлайн оформления билета
   - Пример: `TKTL 10DEC/1200`

5. **Received from (источник бронирования)**
   - Кто создал бронирование
   - Пример: `RF MARIA`

**Дополнительные элементы:**

| Элемент | Назначение | Пример |
|---------|------------|--------|
| SSR DOCS | Паспортные данные | SSR DOCS KC HK1 P/KZ/N12345678/KZ/15MAR85/M/20DEC30/IVANOV/ALEXEY |
| SSR FQTV | Номер бонусной программы | SSR FQTV KC HK1 KC12345678 |
| OSI | Информация для авиакомпании | OSI KC VIP PASSENGER |
| Remarks | Внутренние заметки агентства | RM*CORP CLIENT ABC |

### Ticket issuance

Выписка билета — процесс создания официального документа на перевозку (E-ticket) на основании подтверждённого бронирования.

**Этапы выписки билета:**

1. **Проверка бронирования**
   - Корректность имени (соответствие паспорту)
   - Актуальность рейсов и статусов
   - Наличие всех обязательных элементов

2. **Выбор формы оплаты**
   - Cash (наличные/банковский перевод)
   - Credit Card (кредитная карта)
   - Invoice (выставление счёта корпоративному клиенту)

3. **Расчёт стоимости**
   - Базовый тариф
   - Таксы и сборы (YQ, YR, аэропортовые)
   - Топливный сбор

4. **Выписка и сохранение**
   - Генерация номера билета (13 цифр)
   - Отправка маршрут-квитанции клиенту
   - Сохранение копии в системе

**Структура номера электронного билета:**

```
465 2345678901 2
│   │          │
│   │          └── Контрольная цифра
│   └── Серийный номер (10 цифр)
└── Код авиакомпании (3 цифры, например 465 = Air Astana)
```

**Важные правила выписки:**

- Имя в билете должно точно соответствовать паспорту
- Билет выписывается только на подтверждённые сегменты (статус HK)
- После выписки изменение имени невозможно
- Храните копии всех выписанных билетов

### Reissue

Reissue (переоформление) — изменение условий уже выписанного билета с сохранением его номера или выпуском нового.

**Типы изменений:**

| Изменение | Возможность | Условия |
|-----------|-------------|---------|
| Дата вылета | Обычно да | В пределах срока действия тарифа |
| Маршрут | Зависит от тарифа | Может требовать доплату |
| Класс бронирования | Да | Upgrade/downgrade с пересчётом |
| Имя пассажира | Нет | Требуется новый билет |

**Процесс переоформления:**

1. Проверить правила тарифа (fare rules)
2. Рассчитать разницу в стоимости
3. Учесть сбор за изменение (change fee)
4. Выполнить reissue в системе
5. Отправить новую маршрут-квитанцию

**Расчёт при переоформлении:**

```
Новая стоимость билета
- Стоимость оригинального билета
+ Сбор за изменение
──────────────────────
= Сумма доплаты/возврата
```

**Особенности:**

- Non-refundable билеты при переоформлении сохраняют статус
- ADM (Agent Debit Memo) при ошибках переоформления
- История изменений сохраняется в PNR

### Refunds

Refund (возврат) — аннулирование билета с возвратом денежных средств клиенту.

**Типы возвратов:**

1. **Voluntary refund** — добровольный отказ пассажира
   - Применяются штрафы согласно правилам тарифа
   - Non-refundable тарифы: возврат только такс

2. **Involuntary refund** — вынужденный возврат
   - Отмена/задержка рейса авиакомпанией
   - Полный возврат без штрафов
   - Документальное подтверждение причины

**Процедура возврата:**

1. Определить тип возврата (voluntary/involuntary)
2. Проверить правила тарифа на возвратность
3. Рассчитать сумму к возврату
4. Выполнить refund в системе
5. Контролировать поступление средств

**Сроки возврата:**

| Форма оплаты | Срок возврата |
|--------------|---------------|
| Наличные | 7-14 дней |
| Кредитная карта | 1-2 billing цикла |
| Банковский перевод | 14-30 дней |

**Формула расчёта добровольного возврата:**

```
Стоимость билета
- Штраф за возврат (cancellation penalty)
- Неиспользованные таксы (non-refundable)
──────────────────────
= Сумма к возврату
```

## Практическое применение

**Кейс 1: Ошибка в имени**

Клиент Иванов Алексей забронирован как IVANOV/ALEKSEY (с ошибкой). Билет ещё не выписан.
- **Решение**: Исправить имя в PNR до выписки билета

Если билет уже выписан:
- Связаться с авиакомпанией для уточнения возможности коррекции
- При критической ошибке — возврат и новая покупка

**Кейс 2: Изменение даты корпоративного клиента**

Бизнес-пассажир просит перенести рейс Алматы — Москва на день позже.
1. Проверить наличие мест на новую дату
2. Уточнить правила корпоративного тарифа
3. Рассчитать разницу в стоимости + change fee
4. Согласовать с клиентом
5. Выполнить reissue
6. Выставить счёт на доплату

**Кейс 3: Отмена рейса авиакомпанией**

Air Astana отменила рейс KC 871 из-за погодных условий.
1. Связаться с авиакомпанией для подтверждения отмены
2. Предложить клиенту альтернативы (перебронирование/возврат)
3. При выборе возврата — оформить involuntary refund
4. Документировать для отчётности
