## Что такое GDS?

<Callout type="info">
**GDS (Global Distribution System)** — это компьютеризированная сеть, которая обеспечивает транзакции между поставщиками туристических услуг (авиакомпании, отели, прокат авто) и туристическими агентствами в режиме реального времени.
</Callout>

### Экосистема GDS

```
┌─────────────────────────────────────────────────────────────────────┐
│ GDS ЭКОСИСТЕМА │
├─────────────────────────────────────────────────────────────────────┤
│ │
│ ПОСТАВЩИКИ GDS ПОКУПАТЕЛИ │
│ ────────── ───────────── ────────── │
│ │
│ ┌─────────┐ ┌─────────────┐ ┌─────────┐ │
│ │Авиа- │──────────────│ │────────────│Travel │ │
│ │компании │ │ AMADEUS │ │агентства│ │
│ └─────────┘ │ SABRE │ └─────────┘ │
│ │ TRAVELPORT│ │
│ ┌─────────┐ │ │ ┌─────────┐ │
│ │ Отели │──────────────│ Inventory │────────────│Корпора- │ │
│ │ │ │ + Pricing │ │тивные │ │
│ └─────────┘ │ + Booking │ │клиенты │ │
│ │ │ │
│ ┌─────────┐ └─────────────┘ ┌─────────┐ │
│ │Rent-a- │ │ │OTA │ │
│ │car │────────────────────┘ │(online) │ │
│ └─────────┘ └─────────┘ │
│ │
└─────────────────────────────────────────────────────────────────────┘
```

### Почему GDS важны?

| Без GDS | С GDS |
|---------|-------|
| Звонок в каждую авиакомпанию отдельно | Поиск по всем авиакомпаниям за секунды |
| Ручное сравнение цен | Автоматическое сравнение тарифов |
| Бумажные билеты | Электронное оформление |
| Часы на одно бронирование | Минуты на сложный маршрут |
| Высокий риск ошибок | Стандартизированные процессы |

## История развития GDS

### Временная шкала

```
1960 1970 1980 1990 2000 2020
 │ │ │ │ │ │
 ▼ ▼ ▼ ▼ ▼ ▼
┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐
│SABRE│ │Apollo│ │Amadeus│ │Galileo│ │Слияния│ │NDC │
│(AA) │ │(UA) │ │(EU) │ │(UA) │ │и OTA │ │API │
└────┘ └────┘ └────┘ └────┘ └────┘ └────┘
 │ │ │ │ │ │
 └───────────┴───────────┴───────────┴───────────┴───────────┘
 ЭВОЛЮЦИЯ ДИСТРИБУЦИИ
```

**1960-е**: American Airlines создает SABRE — первую автоматизированную систему бронирования

**1970-е**: United Airlines запускает Apollo, начинается конкуренция

**1987**: Европейские авиакомпании создают Amadeus для независимости от американских систем

**1990-е**: Galileo объединяет несколько систем; появляются OTA (Expedia, Travelocity)

**2000-е**: Консолидация рынка — Travelport поглощает Galileo и Worldspan

**2020-е**: NDC (New Distribution Capability) меняет правила игры

## Amadeus

<Callout type="tip">
Amadeus — основная GDS в Otrar Travel. Изучение команд Amadeus — приоритетная задача для всех сотрудников отдела авиа.
</Callout>

### Ключевые преимущества

- **Глобальный охват**: более 190 стран
- **Полный контент**: интеграция с 490+ авиакомпаниями
- **Amadeus Selling Platform Connect**: современный веб-интерфейс
- **Master Pricer**: мощный поиск тарифов
- **Интеграция с Air Kazakhstan, SCAT, FlyArystan**

### Базовые команды Amadeus

```
┌─────────────────────────────────────────────────────────────────┐
│ CRYPTIC COMMANDS AMADEUS │
├─────────────────────────────────────────────────────────────────┤
│ │
│ ПОИСК И НАЛИЧИЕ (AVAILABILITY) │
│ ───────────────────────────────── │
│ AN15JANALATY → Наличие на 15 января Алматы │
│ AN20FEBTSEDXB → Наличие 20 февраля Астана-Дубай │
│ │
│ БРОНИРОВАНИЕ (SELL) │
│ ────────────────── │
│ SS2Y1 → Забронировать 2 места в классе Y, │
│ сегмент 1 │
│ │
│ ПАССАЖИР (NAME) │
│ ─────────────── │
│ NM1IVANOV/IVAN MR → Добавить пассажира │
│ │
│ КОНТАКТ │
│ ─────── │
│ APM77012345678 → Мобильный телефон │
│ APE-TEST@MAIL.KZ → Email пассажира │
│ │
│ ТИКЕТИНГ │
│ ───────── │
│ TKOK → Разрешение на выписку │
│ TKXL15JAN → Таймлимит до 15 января │
│ │
│ СОХРАНЕНИЕ │
│ ────────── │
│ ER → End and Retrieve (сохранить PNR) │
│ ET → End Transaction │
│ │
│ ОТОБРАЖЕНИЕ │
│ ─────────── │
│ RT → Retrieve (показать текущий PNR) │
│ RTABC123 → Открыть PNR по локатору │
│ │
└─────────────────────────────────────────────────────────────────┘
```

### Пример полного бронирования

```
AN20JANALATSVO ← Поиск рейсов Алматы-Москва на 20 января
SS1Y1 ← Бронируем 1 место в экономе на рейс 1
NM1TURSYNOV/AIBEK MR ← Добавляем пассажира
APM77771234567 ← Добавляем телефон
APE-AIBEK@COMPANY.KZ ← Добавляем email
TKOK ← Подтверждаем возможность выписки
RF OTRAR TRAVEL ← Received From (кто создал)
ER ← Сохраняем бронирование
```

**Результат**: Система возвращает 6-значный код бронирования (PNR), например: `ABC123`

## Sabre

### Особенности системы

- **Sabre Red 360**: Современный графический интерфейс
- **Сильная позиция**: Северная и Латинская Америка
- **GetThere**: Корпоративный модуль самообслуживания
- **Интеграция**: American Airlines, JetBlue, латиноамериканские перевозчики

### Сравнение синтаксиса Amadeus vs Sabre

| Операция | Amadeus | Sabre |
|----------|---------|-------|
| Поиск рейсов | `AN15JANALAIST` | `115JANALAIST` |
| Бронирование | `SS2Y1` | `02Y1` |
| Добавить имя | `NM1IVANOV/IVAN MR` | `-IVANOV/IVAN MR` |
| Телефон | `APM77012345678` | `9MOB77012345678` |
| Сохранить PNR | `ER` | `E` или `ER` |
| Показать PNR | `RT` | `*A` |

<Callout type="warning">
Синтаксис Sabre и Amadeus существенно различается. Не путайте команды при работе в разных системах!
</Callout>

## Архитектура и поток данных GDS

### Как работает бронирование

```
┌─────────────────────────────────────────────────────────────────────┐
│ ПРОЦЕСС БРОНИРОВАНИЯ │
├─────────────────────────────────────────────────────────────────────┤
│ │
│ 1. ЗАПРОС │
│ ┌─────────┐ │
│ │ Агент │ ──→ "Найти рейс ALA-IST на 15 января" │
│ └─────────┘ │
│ │ │
│ ▼ │
│ 2. ПОИСК В GDS │
│ ┌─────────────────────────────────────────┐ │
│ │ GDS СЕРВЕР │ │
│ │ ┌─────────┐ ┌─────────┐ ┌─────────┐ │ │
│ │ │Inventory│ │ Pricing │ │ Rules │ │ │
│ │ └─────────┘ └─────────┘ └─────────┘ │ │
│ └─────────────────────────────────────────┘ │
│ │ │
│ ▼ │
│ 3. ЗАПРОС К АВИАКОМПАНИЯМ │
│ ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐ │
│ │Turkish │ │ SCAT │ │Pegasus │ │ Qazaq │ │
│ │Airlines│ │ │ │ │ │ Air │ │
│ └────────┘ └────────┘ └────────┘ └────────┘ │
│ │ │ │ │ │
│ └──────────┴──────────┴──────────┘ │
│ │ │
│ ▼ │
│ 4. АГРЕГАЦИЯ РЕЗУЛЬТАТОВ │
│ ┌─────────────────────────────────────────┐ │
│ │ Рейс 1: TK 352 ALA-IST 08:00 $450 │ │
│ │ Рейс 2: KC 931 ALA-IST 14:30 $380 │ │
│ │ Рейс 3: PC 752 ALA-IST 22:15 $290 │ │
│ └─────────────────────────────────────────┘ │
│ │ │
│ ▼ │
│ 5. БРОНИРОВАНИЕ │
│ ┌─────────┐ │
│ │ PNR │ ──→ Уникальный код: ABC123 │
│ │ создан │ Статус: Confirmed (HK) │
│ └─────────┘ │
│ │
└─────────────────────────────────────────────────────────────────────┘
```

### Статусы сегментов

| Код | Значение | Описание |
|-----|----------|----------|
| HK | Confirmed | Место подтверждено |
| HL | Waitlist | В листе ожидания |
| HN | Pending | Ожидает ответа авиакомпании |
| UC | Unable to Confirm | Не удалось подтвердить |
| UN | Unable | Сегмент недоступен |
| XX | Cancelled | Отменено |

## Практические упражнения

### Упражнение 1: Поиск рейса

**Задание**: Найдите рейсы из Алматы в Дубай на 25 марта

<details>
<summary>Показать решение</summary>

```
AN25MARALADXB
```

**Расшифровка:**
- `AN` — команда Availability
- `25MAR` — дата
- `ALA` — код IATA Алматы
- `DXB` — код IATA Дубая

</details>

### Упражнение 2: Создание бронирования

**Задание**: Создайте бронирование для пассажира Сергей Ким на рейс в экономическом классе

<details>
<summary>Показать решение</summary>

```
SS1Y1 ← Бронируем 1 место в классе Y
NM1KIM/SERGEY MR ← Добавляем имя
APM77077123456 ← Телефон
APE-SERGEY.KIM@MAIL.KZ ← Email
TKOK ← Тайм-лимит OK
RF OTRAR ← Received From
ER ← Сохраняем
```

</details>

### Упражнение 3: Интерпретация статуса

**Задание**: Что означает статус `HL` в сегменте рейса?

<details>
<summary>Показать ответ</summary>

**HL = Waitlist (Лист ожидания)**

Это означает, что в выбранном классе нет свободных мест, но запрос поставлен в очередь. Авиакомпания может подтвердить место при появлении доступности.

**Действия агента:**
1. Проверять статус регулярно (`RT` для обновления PNR)
2. Предложить клиенту альтернативный класс или рейс
3. Установить напоминание о тайм-лимите

</details>

## Глоссарий

| Термин | Определение |
|--------|-------------|
| **GDS** | Global Distribution System — глобальная дистрибутивная система |
| **PNR** | Passenger Name Record — запись о бронировании пассажира |
| **BSP** | Billing and Settlement Plan — система взаиморасчетов IATA |
| **IATA** | International Air Transport Association — Международная ассоциация воздушного транспорта |
| **Cryptic** | Командный язык GDS (текстовые команды) |
| **Availability** | Наличие мест на рейсе |
| **Segment** | Отдельный перелет в маршруте |
| **E-ticket** | Электронный билет |
| **SSR** | Special Service Request — специальный запрос (питание, багаж, помощь) |
| **OSI** | Other Service Information — дополнительная информация для авиакомпании |
| **NDC** | New Distribution Capability — новый стандарт дистрибуции IATA |
