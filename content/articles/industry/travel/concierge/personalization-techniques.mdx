## Введение

В эпоху онлайн-бронирований персонализированный сервис — главное конкурентное преимущество турагентства. Клиенты могут купить билет самостоятельно, но обращаются к Otrar Travel за индивидуальным подходом и экспертизой.

Для сотрудников Otrar Travel навыки персонализации напрямую влияют на удержание клиентов и увеличение среднего чека. Клиент, чьи предпочтения помнят и учитывают, готов платить премию за сервис и рекомендовать компанию другим.

## Ключевые понятия

### Client profiling

Профилирование клиента — систематический сбор и анализ информации для создания персонализированного опыта.

**Категории данных для профиля:**

| Категория | Что фиксируем | Как используем |
|-----------|---------------|----------------|
| Демография | Возраст, семейное положение, профессия | Выбор категории отелей, активностей |
| Путешествия | Цели поездок, частота, бюджет | Рекомендации направлений, тарифов |
| Предпочтения | Авиакомпании, классы, типы отелей | Приоритет при поиске |
| Ограничения | Диета, здоровье, фобии | Специальные запросы |
| История | Прошлые поездки, отзывы | Избегание повторений, улучшение |

**Структура профиля клиента:**

```
Профиль: Иванов Алексей Петрович

БАЗОВАЯ ИНФОРМАЦИЯ
├── Должность: CFO, ТОО "АБВ"
├── Частота поездок: 3-4 в месяц
├── Основные направления: Москва, Дубай, Франкфурт
└── Предпочитаемый бюджет: бизнес-класс

ПРЕДПОЧТЕНИЯ В АВИА
├── Авиакомпания: Air Astana (Nomad Club Platinum)
├── Место: проход, передние ряды
├── Питание: без свинины
└── Ancillaries: лаунж доступ, приоритетный багаж

ПРЕДПОЧТЕНИЯ В ОТЕЛЯХ
├── Сеть: Marriott (Bonvoy Platinum)
├── Категория: 5*, бизнес-отели
├── Тип номера: King bed, высокий этаж
└── Особые пожелания: тихий номер, кофемашина

ОСОБЫЕ ЗАМЕТКИ
├── День рождения: 15 марта
├── Не любит: длинные пересадки (>3 часов)
└── VIP-фактор: любит комплименты от отелей
```

**Методы сбора информации:**

1. **Прямой запрос** — анкета при первом обращении
2. **Наблюдение** — анализ прошлых бронирований
3. **Обратная связь** — после каждой поездки
4. **Социальные сети** — публичная информация

### Preference tracking

Отслеживание предпочтений — непрерывный процесс обновления профиля клиента на основе его действий и обратной связи.

**Система трекинга предпочтений:**

| Событие | Что фиксируем | Где храним |
|---------|---------------|------------|
| Бронирование | Выбор из предложенных опций | CRM, карточка клиента |
| Изменение | Причина и характер изменений | История бронирований |
| Жалоба | Суть проблемы, решение | Заметки профиля |
| Комплимент | Что понравилось | Поле «позитивный опыт» |
| Отказ | От чего отказался, причина | Поле «избегать» |

**Автоматизация отслеживания:**

```
Триггер: Клиент отказался от предложенного отеля Hilton
         в пользу Marriott

Действие:
1. Зафиксировать в профиле: "Предпочитает Marriott перед Hilton"
2. При следующем поиске: Marriott в приоритете
3. Через 3 подобных случая: Автоматически исключить Hilton
```

**Индикаторы изменения предпочтений:**

- Клиент просит изменить класс обслуживания
- Новые направления, нехарактерные для профиля
- Изменение бюджетных ограничений
- Запрос услуг, которые раньше не использовал

**Best practices отслеживания:**

1. Обновляйте профиль после каждой значимой поездки
2. Спрашивайте обратную связь в правильный момент (24-48ч после возвращения)
3. Фиксируйте не только факты, но и эмоции клиента
4. Делитесь инсайтами с коллегами (при передаче клиента)

### Surprise & delight

Surprise & delight — стратегия превышения ожиданий клиента через неожиданные позитивные действия.

**Пирамида клиентского опыта:**

```
        /\
       /  \
      / WOW \      ← Surprise & delight (5%)
     /______\
    /        \
   / ОТЛИЧНО  \    ← Превышение ожиданий (20%)
  /____________\
 /              \
/ СООТВЕТСТВИЕ   \  ← Базовые ожидания (75%)
\________________/
```

**Категории «сюрпризов»:**

1. **Информационные** (бесплатно)
   - Предупреждение о пробках к аэропорту
   - Информация о погоде в пункте назначения
   - Рекомендации ресторанов рядом с отелем

2. **Сервисные** (низкая стоимость)
   - Апгрейд номера (если есть возможность)
   - Ранний check-in / поздний check-out
   - Лаунж-доступ как комплимент

3. **Материальные** (затраты)
   - Подарок к дню рождения
   - Комплимент в номер
   - Трансфер вместо такси

**Матрица выбора «сюрприза»:**

| Ценность клиента | Повод | Рекомендуемый сюрприз |
|------------------|-------|----------------------|
| Высокая | День рождения | Апгрейд + подарок в номер |
| Высокая | Юбилейная поездка | VIP-встреча + комплимент |
| Средняя | Первая поездка с нами | Информационный пакет |
| Средняя | Длинная задержка рейса | Лаунж-доступ |
| Любая | Проблема по нашей вине | Компенсация + извинения |

**Правила применения:**

1. **Неожиданность** — клиент не должен знать заранее
2. **Релевантность** — сюрприз должен соответствовать профилю
3. **Своевременность** — в правильный момент поездки
4. **Искренность** — не механическое действие, а реальная забота
5. **Документирование** — фиксируйте реакцию для будущих инсайтов

## Практическое применение

**Кейс 1: Первое бронирование нового клиента**

Новый корпоративный клиент запросил билеты Алматы — Москва для руководителя.

**Действия по персонализации:**
1. При приёме заказа задать дополнительные вопросы:
   - «Есть ли предпочтения по авиакомпании?»
   - «Какое место предпочитает — окно или проход?»
   - «Есть ли ограничения по питанию?»
2. Зафиксировать ответы в CRM
3. После поездки — запросить обратную связь
4. Обновить профиль на основе feedback

**Кейс 2: День рождения постоянного клиента**

Постоянный клиент летит в командировку, дата совпадает с его днём рождения.

**Действия по Surprise & delight:**
1. За 3 дня до поездки — связаться с отелем
2. Организовать комплимент в номер (торт/шампанское)
3. Запросить апгрейд номера при возможности
4. В день вылета отправить поздравительное сообщение
5. После поездки — отправить персональную открытку от команды

**Кейс 3: Выявление скрытых предпочтений**

Клиент три раза подряд менял предложенные отели на другие.

**Анализ и действия:**
1. Проанализировать паттерн замен:
   - Первый раз: Hilton → Marriott
   - Второй раз: Sheraton → Ritz-Carlton
   - Третий раз: Holiday Inn → Marriott
2. Выявить закономерность: клиент предпочитает Marriott/Ritz
3. Обновить профиль: «Сеть отелей: Marriott Bonvoy (включая Ritz)»
4. При следующем бронировании предложить только отели сети
5. Спросить напрямую: «Замечаю, что вам нравятся отели Marriott. Есть ли особая причина?»
