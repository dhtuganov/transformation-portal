## Введение

Авиационная индустрия переживает глубокую трансформацию в сфере дистрибуции контента. Традиционные GDS-системы, служившие основой продаж авиабилетов десятилетиями, уступают место новым технологиям прямого подключения. NDC (New Distribution Capability) — стандарт IATA, разработанный для модернизации способа продажи авиаперевозок через агентства.

Для сотрудников Otrar Travel понимание NDC критически важно: авиакомпании активно внедряют этот стандарт, и клиенты ожидают доступа к полному контенту перевозчиков, включая специальные тарифы и дополнительные услуги, недоступные через традиционные каналы.

## Ключевые понятия

### NDC standard

NDC (New Distribution Capability) — стандарт обмена данными на основе XML, разработанный IATA для создания единого «языка» между авиакомпаниями и продавцами.

**Ключевые характеристики NDC:**

- **Rich Content** — передача полного контента авиакомпании: описания продуктов, изображения, брендированные тарифы
- **Real-time Pricing** — динамическое ценообразование в режиме реального времени
- **Персонализация** — индивидуальные предложения на основе истории клиента
- **Ancillaries** — продажа дополнительных услуг (багаж, питание, выбор места) в едином потоке бронирования

**Уровни сертификации NDC:**

| Уровень | Возможности |
|---------|-------------|
| Level 1 | Базовый поиск и бронирование |
| Level 2 | Servicing (изменения, отмены) |
| Level 3 | Полный цикл включая оплату |
| Level 4 | One Order — единый заказ |

### Direct connects

Direct connect — прямое подключение к системам авиакомпании, минуя посредников.

**Типы прямых подключений:**

1. **API авиакомпании** — прямая интеграция через программный интерфейс перевозчика
2. **Агрегаторы NDC** — платформы, объединяющие контент нескольких авиакомпаний (Travelfusion, Farelogix)
3. **Гибридные решения** — комбинация GDS и NDC через единый интерфейс

**Преимущества прямых подключений:**

- Доступ к эксклюзивному контенту
- Более низкие сервисные сборы
- Актуальные остатки мест и цены
- Быстрое подтверждение бронирования

**Вызовы:**

- Необходимость поддержки множества интеграций
- Отсутствие единого интерфейса сравнения
- Различия в процедурах обслуживания

### API integration

API (Application Programming Interface) — набор протоколов для взаимодействия между системами.

**Архитектура NDC API:**

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Otrar Travel   │────▶│   NDC Gateway   │────▶│  Airline System │
│    Frontend     │◀────│   (Aggregator)  │◀────│   (Offer/Order) │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

**Основные методы NDC API:**

| Метод | Описание |
|-------|----------|
| AirShopping | Поиск доступных предложений |
| OfferPrice | Получение финальной цены |
| OrderCreate | Создание бронирования |
| OrderRetrieve | Получение информации о заказе |
| OrderChange | Изменение бронирования |
| OrderCancel | Отмена заказа |

**Технические требования интеграции:**

- Поддержка REST/SOAP протоколов
- Обработка XML/JSON форматов
- Управление сессиями и токенами
- Обработка ошибок и таймаутов

## Практическое применение

**Кейс 1: Бронирование с дополнительными услугами**

Клиент запрашивает перелёт Алматы — Франкфурт с Lufthansa, включая:
- Выбор места у окна
- Дополнительный багаж 23 кг
- Бизнес-лаунж в транзите

Через традиционный GDS эти услуги бронируются отдельно. Через NDC — единый заказ с гарантированной стоимостью всего пакета.

**Кейс 2: Корпоративные тарифы**

Крупный клиент имеет корпоративное соглашение с Air Astana. NDC позволяет автоматически применять согласованные скидки и условия, недоступные через GDS.

**Кейс 3: Динамические предложения**

При поиске рейса система NDC может предложить:
- Альтернативный рейс на 2 часа раньше со скидкой 15%
- Upgrade до бизнес-класса за фиксированную доплату
- Комплект «Путешественник» (багаж + питание + выбор места)

**Рекомендации для агентов:**

1. Всегда проверяйте NDC-контент параллельно с GDS
2. Информируйте клиентов о доступных опциях
3. Документируйте различия в ценах и услугах
4. Используйте NDC для complex itineraries с ancillaries
