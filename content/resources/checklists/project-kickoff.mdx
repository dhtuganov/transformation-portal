# Чек-лист: Запуск проекта

Правильный старт проекта закладывает фундамент для успешной реализации. Этот чек-лист поможет ничего не упустить на этапе kickoff.

## Определение проекта

### Цели и результаты

- [ ] Сформулировать цель проекта (SMART-формат)
- [ ] Определить измеримые критерии успеха
- [ ] Описать ожидаемые результаты (deliverables)
- [ ] Согласовать цели с заказчиком/спонсором
- [ ] Убедиться, что цель понятна всей команде

### Границы проекта (scope)

- [ ] Определить, что входит в проект
- [ ] Зафиксировать, что НЕ входит в проект
- [ ] Описать ключевые допущения (assumptions)
- [ ] Выявить ограничения (constraints)
- [ ] Согласовать scope с ключевыми stakeholders

> **Важно**: Четкие границы проекта предотвращают scope creep (расползание целей).

## Stakeholders (заинтересованные стороны)

### Идентификация

- [ ] Составить список всех заинтересованных сторон
- [ ] Определить заказчика/спонсора проекта
- [ ] Выявить ключевых decision makers
- [ ] Понять, кто будет влиять на проект
- [ ] Определить конечных пользователей результата

### Анализ и коммуникация

- [ ] Провести анализ stakeholders (влияние, интерес)
- [ ] Понять ожидания каждой группы
- [ ] Определить формат коммуникации для каждой группы
- [ ] Согласовать частоту отчетности
- [ ] Создать матрицу коммуникаций

## Команда проекта

### Формирование

- [ ] Определить необходимые роли
- [ ] Назначить руководителя проекта
- [ ] Подобрать членов команды
- [ ] Согласовать загрузку с функциональными руководителями
- [ ] Определить внешних исполнителей (если нужны)

### Организация работы

- [ ] Провести kickoff-встречу с командой
- [ ] Распределить роли и зоны ответственности
- [ ] Установить правила взаимодействия
- [ ] Настроить инструменты коммуникации
- [ ] Создать рабочее пространство (физическое/цифровое)

## Планирование

### Декомпозиция работ (WBS)

- [ ] Разбить проект на фазы/этапы
- [ ] Декомпозировать этапы на задачи
- [ ] Определить связи между задачами
- [ ] Оценить трудозатраты по каждой задаче
- [ ] Выявить критический путь

### Сроки и вехи

- [ ] Установить дату начала проекта
- [ ] Определить целевую дату завершения
- [ ] Установить ключевые вехи (milestones)
- [ ] Построить календарный план (Gantt chart)
- [ ] Согласовать график с командой и заказчиком

### Ресурсы

- [ ] Определить необходимые человеческие ресурсы
- [ ] Рассчитать бюджет проекта
- [ ] Определить необходимое оборудование/инструменты
- [ ] Согласовать бюджет со спонсором
- [ ] Получить доступ к необходимым ресурсам

## Риски и проблемы

### Идентификация рисков

- [ ] Провести сессию по выявлению рисков с командой
- [ ] Составить реестр рисков
- [ ] Оценить вероятность и влияние каждого риска
- [ ] Приоритизировать риски (матрица рисков)
- [ ] Разработать стратегии реагирования

### План управления рисками

- [ ] Назначить ответственных за мониторинг рисков
- [ ] Определить триггеры для активации планов реагирования
- [ ] Подготовить резервные планы (contingency)
- [ ] Заложить резервы времени и бюджета
- [ ] Установить процесс регулярного пересмотра рисков

## Процессы и инструменты

### Управление проектом

- [ ] Выбрать методологию (Waterfall, Agile, Hybrid)
- [ ] Настроить систему управления проектами (Jira, Asana, MS Project)
- [ ] Определить процесс управления изменениями
- [ ] Установить процесс контроля качества
- [ ] Определить формат отчетности

### Коммуникация

- [ ] Создать план коммуникаций
- [ ] Установить регулярные встречи команды
- [ ] Настроить каналы связи (чаты, email-группы)
- [ ] Определить формат статус-репортов
- [ ] Создать хранилище документации

### Документация

- [ ] Подготовить устав проекта (Project Charter)
- [ ] Создать план управления проектом
- [ ] Оформить техническое задание / требования
- [ ] Настроить систему контроля версий документов
- [ ] Получить утверждение ключевых документов

## Kickoff-встреча

### Подготовка

- [ ] Подготовить презентацию проекта
- [ ] Пригласить всех ключевых участников
- [ ] Забронировать место/онлайн-комнату
- [ ] Разослать повестку дня заранее
- [ ] Подготовить материалы для распространения

### Повестка kickoff-встречи

- [ ] Представление участников и их ролей
- [ ] Презентация целей и ожидаемых результатов
- [ ] Обзор плана и ключевых вех
- [ ] Обсуждение рисков и ограничений
- [ ] Утверждение правил работы команды
- [ ] Ответы на вопросы
- [ ] Определение ближайших шагов

### После kickoff

- [ ] Разослать протокол встречи
- [ ] Создать задачи в таск-менеджере
- [ ] Дать доступы всем участникам к инструментам
- [ ] Запланировать первую рабочую встречу
- [ ] Начать мониторинг выполнения первых задач

## Критерии готовности к старту

Проект готов к старту, если:

- ✅ Цели четко сформулированы и согласованы
- ✅ Все ключевые stakeholders идентифицированы
- ✅ Команда сформирована и понимает свои роли
- ✅ Есть утвержденный план и бюджет
- ✅ Риски выявлены и есть планы реагирования
- ✅ Настроены инструменты и процессы
- ✅ Проведен kickoff и зафиксированы договоренности
- ✅ Получены необходимые утверждения и ресурсы

## Типичные ошибки при запуске

- ❌ Нечеткая или неизмеримая цель проекта
- ❌ Отсутствие вовлеченности ключевых stakeholders
- ❌ Недооценка сроков и ресурсов
- ❌ Игнорирование рисков на старте
- ❌ Отсутствие четких ролей в команде
- ❌ Плохо организованный kickoff
- ❌ Нет согласованных процессов коммуникации
- ❌ Отсутствие базового плана для сравнения

> **Правило**: Время, вложенное в планирование на старте, многократно окупается на этапе реализации.
